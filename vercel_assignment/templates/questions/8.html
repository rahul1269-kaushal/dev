Hello [Customer Name], I appreciate you reaching out to Vercel. I understand that you're concerned about your deployment being indexed by search engines. By default, Vercel Preview Deployments are not indexed by search engines because the X-Robots-Tag HTTP header is set to "noindex." However, it's worth noting that if you're using a Custom Domain assigned to a non-Production Branch, the X-Robots-Tag header will not be set to "noindex." To verify the value for the X-Robots-Tag header, you can use the command: curl -I . This command will provide the output of the API call, showing you the header value. If it's not set to "noindex," you can modify it by injecting the response header using your framework's built-in methods. For further insights on indexing and how Vercel Preview Deployments are handled by search engines, you may find this article helpful: https://vercel.com/guides/are-vercel-preview-deployment-indexed-by-search-engines.
